# Servo SDKs

Client SDKs for the Servo backend inference router, available in both JavaScript/TypeScript and Python.

## 📁 Folder Structure

```
SDK/
├── NPM SDK/              # JavaScript/TypeScript SDK package
├── NPM SDK Test/         # Test suite for NPM SDK
├── Python SDK/           # Python SDK package
└── Python SDK Test/      # Test suite for Python SDK
```

## 🚀 Quick Start

### Prerequisites

**Backend must be running:**
```powershell
cd ../inference
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```

### JavaScript/TypeScript

**1. Build the SDK:**
```powershell
cd "NPM SDK"
npm install
npm run build
```

**2. Run tests:**
```powershell
cd "../NPM SDK Test"
npm install
node test.js           # JavaScript test
npx ts-node test.ts    # TypeScript test
```

**3. Use in your project:**
```bash
npm install path/to/NPM-SDK
```

```javascript
const { Servo } = require('servo-sdk');

const client = new Servo({
  baseUrl: 'http://localhost:8000',
  defaultUserId: 'default_user'
});

const result = await client.send('What is machine learning?');
console.log(result.llm_response);
```

### Python

**1. Install the SDK:**
```powershell
cd "Python SDK"
pip install -e .
```

**2. Run tests:**
```powershell
cd "../Python SDK Test"
python test_basic.py
python test_conversation.py
```

**3. Use in your project:**
```python
from servo_sdk import Servo

client = Servo(
    base_url="http://localhost:8000",
    default_user_id="default_user"
)

result = client.send("What is machine learning?")
print(result.llm_response)
```

## 📦 SDK Packages

### NPM SDK

**Location:** `NPM SDK/`

TypeScript/JavaScript client for the Servo backend.

**Key Features:**
- ✅ Full TypeScript type definitions
- ✅ ES modules + CommonJS support
- ✅ Zero dependencies (uses native fetch)
- ✅ Conversation context management

- `src/` - Source code (TypeScript)
- `examples/` - Usage examples
- `dist/` - Compiled output (generated by `npm run build`)

**Documentation:** See [NPM SDK/README.md](NPM%20SDK/README.md)

## 🔧 API Overview

Both SDKs provide identical functionality:

### Core Methods

| Method | Description |
|--------|-------------|
| `health()` | Check backend health status |
| `categories(user_id?)` | Get user-specific category configurations |
| `tiers(user_id?)` | Get category → model mapping |
| `classify(prompt, user_id?, use_quick?)` | Classify a prompt into a category |
| `route(classification, user_id?)` | Route to appropriate model |
| `send(prompt, user_id?, options?)` | Full pipeline: classify + route + invoke LLM |

### Key Types

| Type | Description |
|------|-------------|
| `ClassificationResult` | Contains `category_id`, `category_name`, `reasoning`, `confidence` |
| `ClassificationCategory` | Category metadata with provider, endpoint, model_id |
| `ProcessingResult` | Complete result with classification, routing, and `llm_response` |
| `CategoriesResponse` | User profile with available categories |
| `TiersResponse` | Category ID to model ID mapping |

## 📝 Configuration

Both SDKs support the same configuration options:

**NPM SDK:**
```javascript
new Servo({
  apiKey: 'optional-api-key',
  baseUrl: 'http://localhost:8000',  // default
  timeout: 30000,                     // ms, default 30000
  defaultUserId: 'default_user'       // default 'default_user'
})
```

**Python SDK:**
```python
Servo(
    api_key='optional-api-key',
    base_url='http://localhost:8000',     # default
    timeout_s=30.0,                       # seconds, default 30.0
    default_user_id='default_user'        # default 'default_user'
)
```

## 🐛 Troubleshooting

**Import errors in tests:**
- NPM: Run `npm install` in test folder to install the SDK
- Python: Either install SDK with `pip install -e "../Python SDK"` or tests will use `sys.path` manipulation

**Connection errors:**
- Ensure backend is running on `http://localhost:8000`
- Check backend logs for errors

**Type errors (TypeScript):**
- Run `npm run build` in NPM SDK to regenerate type definitions
- Run `npm install` in test folder to get latest types
